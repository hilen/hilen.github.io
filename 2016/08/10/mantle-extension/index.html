Mantle 的增强

Aug 10, 2016 in iOS

MantleExtension 是在 Mantle 框架提供的类型转换方法的基础上做了一层增强，为了解决服务端返回的同一个数据但出现不同类型而导致客户端 model 解析异常的问题。它可以并且不用修改 Mantle 的源码。

在介绍 MantleExtension 的时候，我们先看来下面这个数据文件：response.json:

Manually，手动进行 JSON/Model 转换

上面一段普普通通的 JSON 数据，在通常的业务开发中，我们经常会和 Model 打交道。因为 Objective-C 或者是 Swift 都是强类型语言，所以也许你遇见过类型错误所带来的崩溃问题。如果你仔细看上面的 JSON 数据，你会发现类型几乎都不相同，举个例子：上面 JSON 文件的注释中 // 就是个典型，里面返回的数据类型是完全不一样的。
好的，那么接下来针对 data 字段进行解析，转化成 Model。我们姑且认为第一个字典里的数据格式和类型是正常的，好了，然后你的 Model 可能会这么写:（为了解决解析时候崩溃的问题，你可能会用到这个分类 NSDictionary+JKSafeAccess.h）

大功告成，然后可以愉快的解析啦。但是呢，或许会有些问题：

  • TSTestModel 的方法 - (id)initWithDictionary:(NSDictionary *)dict可以解析数据。然后如果数据类型出现错误，使用上述方式保证了 Model 解析不会崩溃，但是涉及到业务层的使用，也许就会崩溃了。因为一旦 key 拿不到，NSDictionary+SafeAccess.h 分类返回的都是 nil。

  • 这个 JSON 文件是比较简单的，但是一旦你遇见复杂的 JSON 数据格式，比如这个类型，写 Model 也许会是比较累的一件事情。

  • 层级关系比较复杂的时候，获取最中间的小 model 里面的数据也会变得困难。

那么怎么办呢？

接下来也许你会考虑使用 Objective-C 强大的 runtime 功能来实现我们的 Model 数据解析，这个思路很赞。现在开源社区也出现了很多类似的库，比如 Mantle，JSONModel，YYModel，FastEasyMapping，MJExtension 等等。具体可以看看 YY 写的这篇文章 里的内容。
